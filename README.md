## 📘 프로젝트 개요

C4C는 **Code 4 Cure**의 약자로,
공학 계열 진로를 희망하는 조원 2명과 메디컬 계열 진로를 희망하는 조원 2명이 함께 협업하여 제작한
의료 데이터 분석 기반 웹 프로젝트이다.
프로젝트 명에는 코딩을 의미하는 **Code,** 질병 치료를 의미하는 **Cure,**
그리고 총 **4명의 조원이 참여했다는 의미의 숫자 4**가 함께 담겨 있다.

본 프로젝트는 초기 기획 단계에서
**암 유전체 데이터를 시각화하여 웹으로 제공하는 것**을 1차 목표로 설정하였다.
암 유전체 데이터는 구조가 복잡하고 수치 기반 정보가 많아,
단순한 텍스트나 표만으로는 이해하기 어렵다는 한계가 존재하기 때문이다.
이에 따라 데이터를 시각적으로 표현함으로써
의학적 정보의 접근성과 직관성을 높이고자 하였다.

프로젝트를 진행하는 과정에서,
시각화 자료 자체가 또 다른 해석의 어려움을 유발할 수 있다는 문제를 인식하게 되었고,
이를 해결하기 위한 방법으로 **AI 기반 챗봇 기능의 필요성**을 도출하였다.
AI 챗봇을 통해 사용자가 시각화 결과에 대해 질문하고,
그래프와 데이터의 의미를 설명받을 수 있다면
의학 데이터에 대한 이해 장벽을 효과적으로 낮출 수 있을 것이라 판단하였다.
이로 인해 C4C는 단순 시각화 웹에서 확장되어,
**시각화 + AI 기반 해석 보조**라는 구조로 발전하게 되었다.

또한 의학 연구 및 의료 데이터 분석 과정에서 자주 활용되는
**PDF 형태의 논문 및 자료를 요약하여 핵심 정보를 추출하는 기능**을 포함함으로써,
대량의 의료 문서를 효율적으로 이해할 수 있도록 설계하였다.
이 과정에서 공학적 요소인 데이터 처리, 분석, 자동 요약 기술과
메디컬 관점에서의 데이터 해석이 함께 결합되었다.

### ⏱ 프로젝트 진행 기간

C4C 프로젝트는 **4월에 기획 및 개발을 시작하여 12월 27일에 마무리**되었으며,
약 **9개월간** 기획 → 설계 → 구현 → 개선의 과정을 반복하며 진행되었다.
초기에는 데이터 시각화 중심으로 개발이 이루어졌고,
프로젝트 진행 과정에서 AI 챗봇 및 PDF 요약 기능이 단계적으로 추가되었다.

---

C4C 프로젝트는 여러 개의 HTML 파일을 통해 웹 페이지 구조를 구성하고,
Python 파일을 활용하여 데이터 처리, 시각화, 문서 요약 및 AI 관련 로직을 담당하도록 설계하였다.

프로젝트 전체는
**데이터 수집 → 분석 및 처리 → 시각화 → 해석 보조 → 정보 제공**의 흐름을 따르며,
각 구성 요소는 독립적이면서도 하나의 웹 서비스 안에서 유기적으로 연결된다.
이를 통해 C4C는
공학과 의학이 실제 문제 해결 과정에서 어떻게 협업할 수 있는지를
웹 기반 프로젝트 형태로 구현하는 것을 목표로 한다.

## 🎯 제작 배경

의료 분야에서는 방대한 양의 데이터와 연구 자료가 지속적으로 생성되고 있으며,
특히 암 유전체 데이터는 질병의 원인과 치료 방향을 이해하는 데 중요한 정보를 포함하고 있다.
그러나 이러한 데이터는 구조가 복잡하고 전문적인 해석이 요구되어,
비전공자나 초기 학습자에게는 접근 장벽이 높다는 문제가 존재한다.

프로젝트를 기획하는 과정에서,
암 유전체 데이터가 주로 표나 텍스트 중심으로 제공되는 경우가 많아
정보의 흐름과 의미를 직관적으로 파악하기 어렵다는 점에 주목하였다.
이에 따라 **데이터를 시각화하여 웹 환경에서 제공하면 이해도를 높일 수 있을 것**이라는 문제의식에서
C4C 프로젝트가 시작되었다.

하지만 개발을 진행하면서,
시각화된 데이터 또한 해석 능력에 따라 또 다른 어려움이 될 수 있다는 한계를 인식하게 되었다.
그래프와 수치가 무엇을 의미하는지 이해하지 못하면,
시각화 역시 단순한 결과물에 그칠 수 있기 때문이다.
이 문제를 해결하기 위한 방안으로
**사용자의 이해를 보조할 수 있는 AI 기반 챗봇 기능**을 도입하게 되었다.

AI 챗봇은 사용자가 시각화 자료를 보며 생기는 궁금증을 즉각적으로 질문하고,
데이터와 그래프의 의미를 설명받을 수 있도록 하여
의료 데이터에 대한 이해 과정을 능동적으로 지원하는 역할을 한다.
이를 통해 C4C는
단순히 데이터를 보여주는 웹이 아니라,
**데이터를 해석하고 이해하는 과정을 함께 제공하는 웹**을 목표로 하게 되었다.

또한 의료 연구 과정에서 빈번하게 사용되는
PDF 형태의 논문 및 보고서를 효율적으로 분석하기 위해,
핵심 내용을 요약하여 제공하는 기능을 포함하였다.
이는 방대한 문서를 모두 읽어야 하는 부담을 줄이고,
필요한 정보를 빠르게 파악할 수 있도록 하기 위한 설계이다.

이러한 배경 속에서 C4C는
공학적 기술을 활용한 데이터 처리와 시각화,
메디컬 관점에서의 데이터 해석과 의미 도출이 결합된
**융합형 의료 데이터 분석 웹 프로젝트로** 기획·제작되었다.

## 🧠 프로젝트 핵심 컨셉

C4C 프로젝트의 핵심 컨셉은
**“암 유전체 데이터의 이해 장벽을 기술로 낮춘다”** 이다.

이를 위해 프로젝트는 다음의 세 가지 축을 중심으로 설계되었다.

### 1️⃣ 암 유전체 데이터 시각화

복잡한 암 유전체 데이터를
그래프, 차트, 인터랙티브 시각화 형태로 제공함으로써
사용자가 데이터의 패턴과 특징을 직관적으로 파악할 수 있도록 한다.

- 수치 중심 데이터 → 시각적 정보로 변환
- 비교·분포·변화 양상을 한눈에 확인 가능
- 단순 나열이 아닌 해석 가능한 시각화를 목표로 설계

### 2️⃣ AI 챗봇을 통한 데이터 해석 보조

프로젝트 초기에는 데이터 시각화 기능만을 목표로 하였으나,
개인적인 문제의식에서 AI 챗봇 기능이 핵심 요소로 확장되었다.

시각화 자료는 보기에 따라 오히려
“무엇을 의미하는지 모르겠다”는 새로운 어려움을 만들 수 있기 때문에,
AI 챗봇을 통해 다음과 같은 역할을 수행하도록 기획하였다.

시각화 결과에 대한 질문 응답

그래프와 수치가 의미하는 바를 자연어로 설명

사용자의 이해 수준에 맞춘 보조 설명 제공

즉,
시각화 + AI 챗봇 구조를 통해
데이터 해석의 진입 장벽을 낮추는 것을 핵심 컨셉으로 삼았다.

### 3️⃣ 암 관련 PDF 문서 요약 기능

암 연구 논문이나 보고서는 대부분 PDF 형태로 제공되며,

분량이 길고

전문 용어가 많아
학생이 전체를 읽고 핵심을 파악하기 어렵다.

이를 해결하기 위해
PDF 문서를 업로드하면 핵심 내용을 요약해주는 기능을 포함시켜,
사용자가 연구 자료의 핵심만 빠르게 이해할 수 있도록 하였다.
